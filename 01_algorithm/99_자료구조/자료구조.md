# 자료구조란?

데이터의 효율적인 접근 및 조작을 가능하게 해주는 저장 및 관리 방식

여러 상황에 적합한 구조를 고르는 방법

# 상황에 맞는 자료 구조

다음의 경우 세트가 리스트보다 빠르다.

```python
import time

test_list = list([x for x in range(0, 1000001)])
test_set = set([x for x in range(0, 1000001)])

t_0 = time.time()
print(1000000 in test_list)
t_1 = time.time()
print("리스트에서 걸린 시간: {}".format(t_1 - t_0))

t_0 = time.time()
print(1000000 in test_set)
t_1 = time.time()
print("세트에서 걸린 시간: {}".format(t_1 - t_0))
```

```python
리스트에서 걸린 시간: 0.013919830322265625
세트에서 걸린 시간: 3.814697265625e-06
  # 0.000003814697265625
```

- 부동소수점 표시 e-6 이므로 소수점을 왼쪽으로 6칸 이동.
- 100e - 6 = 100/1000000.0f
- 2.4453e9 = 2.4453f * 1000000000f

# 스토리지 vs. 메모리

- 스토리지 : 데이터가 영구적으로 저장되는 곳
  - 데이터를 저장하는 데 오래 걸린다.
  - 데이터를 받아오는 데 오래 걸린다.
  - 정확히 언제 사용할 지 모르는 데이터 저장
- 메모리 : 임시 저장
  - 데이터를 저장하는 데 빠르다.
  - 데이터를 받아오는 데 빠르다.
  - 당장 사용해야 하는 데이터를 저장

=> 자료구조는 데이터를 메모리에서 잘 사용하게 하는 것!!

# 메모리

메모리는 하나의 긴 띠이며 임의 접근으로 동작한다.

1. 일정한 칸으로 나눠져 있다.
2. 각 칸에 데이터를 저장할 수 있다.
3. 각 칸은 자신만의 주소가 있다.

### RAM : 임의 접근 메모리

- 저장 위치를 알면 접근시 항상 일정한 시간이 걸린다. 
- O(1)
- 순차접근보다 임의접근이 효율적이다.

### 바이트(byte)

컴퓨터 저장 공간 용량을 나타내는 단위.

메모리 한 칸이 저장할 수 있는 기본적인 용량의 단위는 1바이트다.

- 킬로 바이트 = 1,000 바이트
- 메가 바이트 = 1,000,000 바이트
- 기가 바이트 = 1,000,000,000 바이트

# 레퍼런스(reference)

- 레퍼런스 : 데이터에 접근할 수 있게 해주는 값
- x = 95
  - x 는 95다 x (95라는 데이터를 담고 있다. x)
  - x 는 95를 가리킨다. (95가 저장된 곳의 레퍼런서를 갖고 있다.)
- x에는 레퍼런스가 담겨있지만, 변수 사용시 파이썬이 메모리에서 값을 받아와서 사용한다.
