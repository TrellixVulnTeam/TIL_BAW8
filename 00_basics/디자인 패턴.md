# Design Pattern

프로그래밍에서 고질적으로 발생하는 문제들을 해결하기 위한 테크닉



## 23가지 디자인 패턴

특정 상황에서 나타나는 고질적인 문제를 구조적으로 해결할 수 있다.

항상 모두 적용해야 하는 것은 아니며, 상황과 필요에 따라서 선택적으로 사용하면 된다.

## 코드 분리

코드의 구조화를 위해서는 의미 있는 코드들을 묶어서 코드 분리를 해야한다.

코드 분리를 하면 프로그램의 기능을 바꾸고 싶을 때 원하는 부분만 변경할 수 있다.
